# JOTO Bot — генератор артикулов и карточек WB

## Что делает бот
1. Принимает сообщение от пользователя в Битрикс24
2. Генерирует артикул по правилам JOTO (J + категория + модель + /цвет)
3. Автоматически создаёт карточку товара на Wildberries

## Формат сообщения боту
```
категория: худи
модель: 3
цвет: black
название: Худи оверсайз мужское
```

Бот ответит: `J05003/black` и создаст карточку на WB.

## Установка на Railway

### Шаг 1 — GitHub
1. Зарегистрируйся на github.com
2. Создай новый репозиторий (New repository), назови `joto-bot`
3. Загрузи все файлы из этой папки

### Шаг 2 — Railway
1. Зайди на railway.app
2. Войди через GitHub
3. New Project → Deploy from GitHub repo → выбери `joto-bot`
4. Railway сам запустит бота

### Шаг 3 — Переменные окружения
В Railway → твой проект → Variables → добавь:
- `WB_API_TOKEN` = твой токен Wildberries
- `B24_WEBHOOK` = https://joto.bitrix24.ru/rest/226/ТВОЙ_КЛЮЧ

### Шаг 4 — Получи URL бота
В Railway → Settings → Domains → Generate Domain
Скопируй URL (например: https://joto-bot.up.railway.app)

### Шаг 5 — Подключи к Битрикс24
1. В Б24 → Разработчикам → Добавить чат-бот → Передавать боту сообщения из чата
2. В поле "URL обработчика" вставь: https://joto-bot.up.railway.app/webhook
3. Сохрани

## Массовые карточки WB (раздел `/cards`)

Отдельный раздел внутри приложения для **массового создания и редактирования карточек на Wildberries**. Менеджер заходит во встроенное приложение Bitrix24 и переключается между двумя режимами — все изменения автоматически уходят на WB через Content API.

Открыть: ссылка **«Массовые карточки WB →»** на главной странице или адрес `/cards`.

### Режим «Редактировать массово»
1. **Загрузить карточки с WB** (можно с фильтром по артикулу/названию).
2. Отметить нужные карточки галочками (есть «выбрать все» и фильтр).
3. Выбрать в выпадающем списке **что менять**: SEO-описание, название, бренд, артикул, категория, характеристика или добавить баркод.
4. Ввести новое значение → **«Применить и сохранить на WB»** (`content/v2/cards/update`).

> Артикул и баркод уникальны для каждой карточки — меняйте их по одной.
> Смену категории у существующей карточки WB разрешает не всегда; при отказе вернётся ошибка WB.

### Режим «Создать массово»
1. Найти и выбрать **категорию WB**, задать общие поля (бренд, габариты, цена, размер).
2. При необходимости загрузить **характеристики категории** и заполнить общие значения.
3. Добавить строки карточек (артикул, название, SEO-описание, баркод). Баркоды можно **сгенерировать** кнопкой.
4. **«Создать карточки на WB»** (`content/v2/cards/upload`).

> Для этого раздела `WB_API_TOKEN` должен иметь доступ к категории **«Контент»** (создаётся в ЛК WB → Настройки → Доступ к API).

### Пункт в левом меню Bitrix24
Приложение регистрирует отдельный пункт **«Карточки WB»** в левом меню Bitrix24 (через `placement.bind` c `LEFT_MENU`). Он появляется автоматически при установке/переустановке приложения.

Если приложение **уже установлено**, привязать пункт меню без переустановки можно сервисным запросом:
```
GET /admin/bitrix/placement?secret=<BITRIX_CLIENT_SECRET>
```
После этого пункт «Карточки WB» появится в левом меню (может потребоваться обновить страницу Bitrix24).

### API раздела
- `GET  /api/wb/cards?search=&limit=` — список карточек
- `POST /api/wb/bulk-edit` — массовое изменение поля у выбранных карточек
- `POST /api/wb/create` — массовое создание карточек
- `POST /api/wb/barcodes` — генерация баркодов
- `GET  /api/wb/subjects?name=` — поиск категорий WB
- `GET  /api/wb/charcs?subjectId=` — характеристики категории
